## 核心流程详解

### 1. 文件上传完整流程

```
用户选择文件
     │
     ▼
┌─────────────────┐
│ 1. 计算文件 MD5  │
│    (SparkMD5)   │
└─────────────────┘
     │
     ▼
┌─────────────────┐
│ 2. 检查秒传     │
│    POST /check  │
└─────────────────┘
     │
     ├─► 文件存在 ──► 返回文件 URL ──► 完成 ✓
     │
     ▼ 文件不存在
┌─────────────────┐
│ 3. 初始化上传   │
│    POST /init   │
│  - 校验大小     │
│  - 计算分片数   │
│  - 创建会话     │
└─────────────────┘
     │
     ▼
┌─────────────────┐
│ 4. 循环上传分片 │
│    POST /chunk  │
│  - 限流检查     │
│  - 接收分片     │
│  - 写入磁盘     │
│  - 更新进度     │
└─────────────────┘
     │
     ├─► 未完成 ──┐
     │            │
     ▼ 已完成     │
┌─────────────────┤
│ 5. 合并分片     │
│    - 排序分片   │
│    - 顺序合并   │
│    - 清理临时   │
│    - 记录 MD5   │
└─────────────────┘
     │
     ▼
┌─────────────────┐
│ 6. 返回结果     │
│    fileUrl      │
└─────────────────┘
```

### 2. 秒传检测流程

```
前端计算 MD5
     │
     ▼
POST /api/upload/check?fileMd5=xxx
     │
     ▼
后端查询 Redis
     │
     ├─► Key 存在 ──► 文件已存在
     │                 │
     │                 ▼
     │            返回 fileUrl
     │                 │
     │                 ▼
     │            前端显示成功 ✓
     │
     ▼ Key 不存在
返回 needContinue=true
     │
     ▼
前端继续分片上传
```

### 3. 分片上传详细流程

```
for (i = 0; i < totalChunks; i++) {
    │
    ▼
┌──────────────────┐
│ 切片: file.slice │
│ start = i * 2MB  │
│ end = (i+1) * 2MB│
└──────────────────┘
    │
    ▼
┌──────────────────┐
│ 构建 FormData    │
│ - fileMd5        │
│ - chunkIndex     │
│ - chunkFile      │
└──────────────────┘
    │
    ▼
┌──────────────────┐
│ POST /chunk      │
│ Content-Type:    │
│ multipart/form-  │
└──────────────────┘
    │
    ▼
┌──────────────────┐
│ 后端处理：       │
│ 1. 限流检查      │
│ 2. 保存到临时目录│
│ 3. Redis +1      │
│ 4. 检查是否完成  │
└──────────────────┘
    │
    ├─► 未完成 ──► 继续下一分片
    │
    ▼ 已完成
┌──────────────────┐
│ 合并所有分片     │
│ 删除临时文件     │
│ 返回 fileUrl     │
└──────────────────┘
}
```

### 4. 限流机制

```
用户发起上传请求
     │
     ▼
┌──────────────────┐
│ 用户维度限流     │
│ RateLimiter      │
│ tryAcquire(0)    │
└──────────────────┘
     │
     ├─► 失败 ──► 返回 429 Too Many Requests
     │
     ▼ 成功
┌──────────────────┐
│ 每日次数检查     │
│ Redis INCR       │
│ daily:{date}:uid │
└──────────────────┘
     │
     ├─► 超限 ──► 返回错误提示
     │
     ▼ 未超限
┌──────────────────┐
│ 全局限流检查     │
│ Guava RateLimiter│
└──────────────────┘
     │
     ├─► 失败 ──► 稍后重试
     │
     ▼ 成功
允许上传
```

## 数据存储设计

### Redis Key 设计

| Key 模式 | 类型 | 用途 | TTL |
|---------|------|------|-----|
| `file:md5:{md5}` | String | 秒传检测，存储 fileUrl | 30 天 |
| `upload:{uploadId}` | Hash | 上传会话元数据 | 7 天 |
| `chunk:uploaded:{md5}` | String | 已上传分片计数 | 7 天 |
| `upload:daily:{date}:{uid}` | String | 每日上传次数 | 1 天 |

### 文件系统结构

```
uploads/
├── chunks/              # 临时分片目录
│   └── {fileMd5}/
│       ├── 0            # 分片 0
│       ├── 1            # 分片 1
│       └── ...          # 更多分片
│
└── files/               # 最终文件目录
    └── 2024/
        └── 01/
            └── 15/
                └── {uuid}_{filename}
```

## 性能优化点

### 1. 内存优化
- 分片接收后立即写入磁盘，不占用堆内存
- 使用流式 MD5 计算，避免一次性加载整个文件
- 合并分片时使用流式拷贝

### 2. 并发控制
- 每个用户独立的 RateLimiter
- 全局限流保护服务器
- Redis 原子操作保证数据一致性

### 3. 存储优化
- MD5 去重节省空间
- 临时文件自动清理
- 按日期分目录存储

### 4. 网络优化
- 分片大小权衡（2MB）
- 支持断点续传减少重复传输
- 秒传机制避免重复上传

## 扩展性设计

### 支持云存储

当前实现基于本地文件系统，可通过实现 Storage 接口扩展：

```java
interface StorageProvider {
    void saveChunk(String md5, int index, InputStream data);
    void mergeChunks(String md5, String fileName);
    String getFileUrl(String md5);
    boolean exists(String md5);
}
```

实现类：
- LocalStorageProvider（当前）
- AliyunOssProvider
- S3StorageProvider

### 支持分布式

Redis Cluster 部署：
```yaml
spring:
  data:
    redis:
      cluster:
        nodes:
          - redis1:6379
          - redis2:6379
          - redis3:6379
```

### 支持微服务

可拆分为独立服务：
- Upload Service（无状态，可水平扩展）
- Merge Service（负责分片合并）
- Storage Service（负责文件存储）
